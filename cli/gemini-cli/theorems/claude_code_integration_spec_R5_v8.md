# Tunnel Calculus: A Formal Framework for Residual Information under Collapse Maps (v5.4 - R5_v8)

## 概要
本仕様書は、自律型AIコーディングエージェントおよびPC上の情報システムに対し、「崩壊写像（Collapse Map）」によって情報を正規化・圧縮しつつ、失われる差分を「残差（Residual）」として保持・復元する計算形式である **「トンネル計算（Tunnel Calculus）」** の数理的定式化およびPC向け応用設計を規定する。

第8周目 (v8) においては、L3（スキル直交性）の混雑度ペナルティ $P_{penalty}(\rho)$ が L5 の期待コスト臨界閾値 $P_{crit}$ に早期干渉してルーティングを阻害する不整合を解消し、中程度の混雑度においても直交回避ロジック（DQS, OPF, HNPT）が完全にその能力を発揮するための「ペナルティのスケーリング調整」を数理的に統合する。

## 1. トンネル計算 (Tunnel Calculus) の 5大数学コンポーネント

トンネル計算は、情報損失と復元のメタ幾何学を記述するため、以下の5つの数理的構成要素によって定義される。

### A. 局所回転骨格 (Local Rotation Skeleton)
局所的な線形変化および直交表現のための基本骨格として、複素構造および回転を以下のように包含する。
$$J = \begin{pmatrix} 0 & -1 \\ 1 & 0 \end{pmatrix}, \qquad J^2 = -I, \qquad e^{\theta J}$$

### B. 崩壊写像 (Collapse Map)
入力空間 $A$ から、代表値（正規化キー）の空間 $B$ （極限としては単一元空間 $\mathbf{1}$）へ、すべての情報を圧縮・正規化して押し潰す写像。
$$C: A \to B$$

### C. トンネル・セクション (Tunnel Section)
潰された代表値 $b \in B$ から、元の多様なファイバーの枝 $\alpha \in C^{-1}(b)$ を指定して元の空間 $A$ へ戻るための非等価な局所復元セクション。
$$t_\alpha: B \to A, \qquad C \circ t_\alpha = \operatorname{id}_B$$

### D. 留数作用素 (Residue Operator)
代表値に押し潰された際、特異点の周回積分によって「潰れずに残る残余成分」を抽出する作用素。
$$T(f) = \frac{1}{2\pi i} \oint_\gamma f(z) \, dz$$

### E. 整数不変量 (Integer Invariant)
関数の巻き数、次数、あるいはエージェントの状態遷移空間における閉路数など、構造が揺らいでも保存される整数の骨格。
$$N(f) = \frac{1}{2\pi i} \oint_\gamma \frac{f'(z)}{f(z)} \, dz \in \mathbb{Z}$$

## 2. PC上での実用プロダクト設計と優先度

PC上でトンネル計算を具現化するにあたり、最も有用な「残差キャッシュ」モデルは以下の通りである。

$$x \mapsto (C(x), \alpha)$$
*   $C(x)$: 正規化された骨 (`canonical key`)
*   $\alpha$: 潰しても残る差分・例外・署名 (`residual branch`)
*   $t_\alpha(C(x))$: 必要な文脈を復元 (`reconstructor`)

### 優先プロダクト設計

#### 1. Tunnel Research Compiler (思考・研究メモの論文化支援)
*   **collapse ($C$)**: ユーザーの雑多な思考メモ、数式、直感的な記述を「定義」「定理」「証明」「未解決」などの学術的スキーマの骨格に自動分類。
*   **residue ($\alpha$)**: 理論の飛躍、危険な過大評価（例: 物理的宇宙問題への無理な接続等）、および未検証の参照（例: 査読の経ていない個人論文の引用等）を残差として自動検出・分離。
*   **invariants ($N$)**: 証明における形式的整合性のリスク度を整数不変量として評価。

#### 2. Tunnel Error Resolver (エラー解決エンジン)
*   **collapse ($C$)**: PC上で発生した多種多様なエラーログから、環境依存情報を除去して `PermissionError` などのエラーの基本形に潰す。
*   **residue ($\alpha$)**: 環境変数、パス、プロセスのPID、依存パッケージバージョンなどの環境固有情報を残差として抽出。
*   **tunnel ($t_\alpha$)**: 残差情報を組み込んで、その環境に特化した解決コマンド（例: `chmod` や `kill` の提案）を生成。

#### 3. Tunnel Clipboard (クリップボード履歴の意味化)
*   **collapse ($C$)**: コピーされたテキストを「URL」「コード」「プレーンテキスト」などに分類。
*   **residue ($\alpha$)**: URLの本体とトラッキングパラメータの分離、あるいは類似する複数回コピーの微小差分を抽出。

## 3. 統合コヒーレンスの方程式系 (v5.4最終定式)

各レイヤーの境界値は、以下の数理的結合方程式を通じて完全同期し、システム全体の調和的安定性を担保する。

1.  **ルーティングと成功確率 (L3 $\to$ L5) ［v8 スケーリング調整適用］**:
    L3ルーティングの曖昧度誤差を $\epsilon(q)$ としたとき、L5における自律タスクの成功確率は以下のように制約される：
    $$P_{success} \le (1 - \epsilon(q)) P_{base}$$
    さらに、混雑度 $\rho = N/N_{max}$ に伴う「予測残余誤り率」を、直交回避ロジック（DQS, OPF, HNPT）適用後の性能を反映して以下のように定義する：
    $$P_{penalty}(\rho) = \eta_{route} \cdot \left( 1 - \frac{1}{\max(\rho, 1e-9)} \right)$$
    ここで、 $\eta_{route} = 0.05$ は直交回避ロジックによる決定不能相の解決能力を示す減衰（圧縮）係数である。この残差ペナルティ見積もり $P_{penalty}(\rho)$ が L5 の臨界閾値 $P_{\text{crit}} = (C_{\text{hitl}} - C_{\text{exec}}) / C_{\text{fail}} = 0.08$ を超えた場合（あるいは $\rho \ge \rho_{threshold} = 2.5$）、システムは自律動作を強制遮断して L5 (HITLエスカレーション) へ安全にフォールバックする。
    これにより、$\rho \ge 1.09$ の初期混雑段階で DQS や OPF の分岐を試す前に早期エスカレーションされてしまう不整合を回避し、自律解決能力を最大化する。

2.  **セキュリティ制限と成功確率 (L4 $\to$ L5)**:
    L4の実効リスク $I_{eff} \ge I_{crit}$ のとき、サンドボックス権限が収縮するため、L5の自律成功確率は強制的に $P_{success} = 0$ となり、直ちにHITL介入閾値を下回ってエスカレーションを発生させる。

3.  **状態解像度とループフリー (L6 $\to$ L2)**:
    L6がコンテキスト窓節約のために忘却を行う際、L2のループフリー判定を保証するために必要な最小マーク解像度 $H_{crit}$ が下限境界として強制される：
    $$\sum_{i: active} \eta(a_i) H_i R_i \ge H_{crit}$$

4.  **デッドラインと遅延境界 (L7 $\to$ L6)**:
    L7のSlack Time $L(t)$ に基づき、L6の許容遅延境界 $D_{max}$ が $D_{max}(t) \propto L(t)$ として動的に短縮される。これに伴い、遅延のシャドウプライス $\mu \to \infty$ となり、スワップおよび圧縮処理が禁止され、保持（Keep）か忘却（Forget）かのバイナリ選択（LIBC: Latency Induced Binary Choice）へと相転移する。このとき、境界条件が崩壊している要素 ($\lambda s_i > R_i$) は純効用が負となるため強制的に $Forget$ される。

5.  **配信遅延と制御ループの応答安定性 (SSE $\to$ L7)**:
    Next.js 15 (SSE) によるステータス配信における伝送遅延の総和を $\tau_{\text{sse}}$ としたとき、遷移チャタリングおよびデッドライン違反を防ぐため、$\tau_{\text{sse}}$ は以下の許容遅延境界を常に下回らなければならない：
    $$\tau_{\text{sse}} < \min_i (T_{\text{dwell}}, L(t))$$

6.  **隔離復帰遅延極限と強制退行 (L4 $\leftrightarrow$ L7)**:
    L4の完全隔離状態 $S(t_{emit}) = 1.0$ に突入した際、L7の残余 Slack Time $L(t_{emit})$ が復元に要する最小遅延時間 $t_{rec}$（人間監査応答とステップ回復遅延の和）を下回る場合、システムはタイムアウトによるハングアップを回避するため、即座に「超縮退運転モード」 $M^{(4)}$ へ強制相転移する：
    $$L(t_{emit}) \le t_{rec} \implies M(t) \to M^{(4)}$$

7.  **自己修復限界と相転移 (L9 $\to$ L7/L5)**:
    L9の自己修復中に純粋なAST意味ハッシュの履歴衝突が検知された場合、または試行イテレーション回数が `MAX_L9_MUTATIONS` (上限 5) に達した場合、遷移インジケータ $P_{transit}(t) = 1$ が成立し、システムは自律修復を直ちに打ち切る。これにより、L2における有向閉路を完全に排除した上で、L7（縮退運転モード）または L5（HITLエスカレーション）へ安全に制御権を相転移させる。

8.  **一般化ポテンシャル関数による膠着状態の一網打尽化 (L2 / L7 / L9 統合)**:
    L2（重複状態履歴 $\mathcal{H}_t$）、L7（残余 Slack Time $L(t)$）、および L9（残余自己修復変異バジェット $k$）を単一の『一般化ポテンシャル関数 $\Psi_{unified}$』として統合し、システム状態を自律制御する。
    $$\Psi_{unified}(v_t, \mathcal{H}_t, L(t), k) = \Phi(v_t) + \alpha_{loop} \cdot \mathbb{I}(v_t \in \mathcal{H}_t) + \gamma_{L7} \cdot \left( \frac{1}{\max(L(t), 1e-9)} \right) + \beta_{L9} \cdot (5 - k)$$
