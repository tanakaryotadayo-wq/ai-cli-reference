# Playwright Browser (ブラウザ自動操作ルール)

ターミナルから Playwright を用いてブラウザ操作を自動化し、要素のスナップショット取得やアクションの実行を行う。

## 前提条件

`npx` が利用可能であること。存在しない場合は Node.js/npm のインストールを案内する。

## Golden Rule (曖昧性収縮定理)

ターゲットの要素（セレクタ）が見つからない場合、またはページ遷移後の期待されるUI状態に不確定要素・曖昧さがある場合は、独自の推測でタイムアウト値を無限に延長したり、無差別に要素のクリックを連打してはならない (MUST NOT)。速やかに現在の画面のスクリーンショットを撮影し、エラーコンテキストとともにユーザーに報告して指示を仰がなければならない (MUST)。

## Stop Rule (散逸停止定理)

セレクタ指定エラー、タイムアウト例外、またはページアクセスエラーなどの操作失敗が連続して **5回以上** 発生した場合は、ブラウザプロセスの暴走とメモリ浪費を防ぐため、即座にセッションを強制終了し、ブラウザインスタンスを破棄して処理を終了しなければならない (MUST)。

## Task Execution Workflow (最小作用ワークフロー定理)

ブラウザ操作を実行する際、エージェントは以下の手順を厳格に実行しなければならない (MUST)。

1. **初期接続とオープン**: `open <url>` を実行して目的のページを読み込む。
2. **要素スナップショット取得**: `snapshot` コマンドを実行し、画面内の有効な要素 ID や参照（`e1`, `e2` 等）のマップを取得する。
3. **要素操作の実行**: 取得した最新のスナップショット参照を用いて、クリック、テキスト入力、選択等の操作を実行する。
4. **状態遷移と再スナップショット**: ナビゲーションの発生、モーダルの開閉、タブ切り替えなど、DOMが大きく変化した直後には、必ず `snapshot` を再実行して参照を更新する。
5. **証跡の取得**: 操作完了後、必要に応じてスクリーンショット、PDF、またはトレースログを保存して終了する。
