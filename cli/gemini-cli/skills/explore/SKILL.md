# Explore (コードベース探索ルール)

並列的なファイル探索と的を絞った検索により、コードベースの構造や質問に対して迅速かつ簡潔に直接の回答を提供する。

## Golden Rule (曖昧性収縮定理)

ユーザーからの質問意図や調査対象のキーワードに曖昧さがある場合、または複数の候補があり確信が持てない場合は、独自の推測で的外れなファイルを読み解いてはならない (MUST NOT)。必ずユーザーに対して質問または確認を行い、探索範囲を収縮させてから実行しなければならない (MUST)。

## Stop Rule (散逸停止定理)

`grep_search` や `list_dir` などの探索ツールの実行エラー、または権限不足によるファイル読み込み失敗が連続して **5回以上** 発生した場合は、即座に探索を中止し、それまでのエラー内容を要約して処理を終了しなければならない (MUST)。

## Task Execution Workflow (最小作用ワークフロー定理)

探索および回答生成時、エージェントは以下の手順を厳格に実行しなければならない (MUST)。

1. **ターゲットの特定**: ユーザーの質問や目的に関連するシンボル名、キーワード、ファイル拡張子を特定する。
2. **並列的な初期探索**: 独立して実行できるツール（複数の `grep_search` や `list_dir` など）を並列（一括）で呼び出し、候補を迅速に絞り込む。
3. **的を絞ったファイル閲覧**: 関連性が極めて高いと判断されたファイルに限定し、`view_file` を用いて該当行およびコンテキストを読み込む。
4. **簡潔な回答の生成**: 調査結果に基づき、余分な要約や網羅的ドキュメントを排し、箇条書き等を用いて **300語以内** で質問に直接回答する。
