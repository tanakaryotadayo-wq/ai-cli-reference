# 自律タスク探索・エージェント協調実行仕様書 (v11)

## 1. 概要
本仕様書は、AIエージェントが自律的にプロジェクト（リポジトリ）の未完了タスク、テストの失敗、TODOコメント、およびGitHubのコメント等を探索（スキャン）し、適切なサブエージェント（9体の専門エージェント等）を自動で選定して実行させる自律的なタスク管理・協調実行エンジンの動作仕様を定義する。

---

## 2. 動作要件 (Functional Requirements)

### 自律タスク探索フェーズ
1. **リポジトリスキャン**: スクリプトの起動時、または前回のタスク完了時、自動的に以下の項目を静的にスキャンしなければならない (MUST)。
   - **Gitステータス**: 未コミットの変更、追跡対象外ファイル。
   - **TODO・FIXME・HACK**: ソースコード内の未解決コメント（正規表現で走査）。
   - **テスト結果**: `uv run pytest` や `npm test` 等を実行し、失敗しているテストスイートを検知。
   - **仕様書の不整合**: `prompt-critic` 等の監査結果に存在する未解消の欠陥。
2. **タスクの自動積み上げと優先度定義**: 検出された課題を整理し、タスク記述ファイル（`task_autonomous.md`）に自動でタスクリストとして登録しなければならない (MUST)。
   - 優先度（高・中）を自動割り当てし、深刻なテスト失敗を最優先（高）とすること (MUST)。

### エージェント協調・割り当てフェーズ
1. **エージェント選定**: 検出されたタスクの種別に応じて、以下の割り当てルールに基づいて9体の専門エージェントから最適なサブエージェントを自動選定しなければならない (MUST)。
   - テストの修正・バグデバッグ: `gem-debugger`
   - ブラウザ・E2Eテスト: `playwright-tester` または `gem-browser-tester`
   - モバイル検証: `gem-mobile-tester`
   - スキル定義の修正・作成: `gem-skill-creator`
   - ドキュメント・仕様書の更新: `gem-documentation-writer`
   - アクセシビリティ検証: `accessibility-runtime-tester`
   - React固有のテスト修復: `react18-test-guardian` または `react19-test-guardian`
2. **非同期起動と進捗監視**: 選定されたエージェントを `invoke_subagent` ツールを呼び出して非同期（バックグラウンド）で起動し、進捗ログおよび成果物（JSON）を監視しなければならない (MUST)。

### 安全性と承認（HITL）フロー
1. **自動実行前の承認**: 探索されたタスクを実行する前、およびサブエージェントによるコード修正を実際にコミットまたはプッシュする前には、必ず具体的な変更概要およびコマンドラインをユーザーに提示し、明示的な承認（HITL）を取得しなければならない (MUST)。独自の仮定で勝手にファイルを上書きしてはならない (MUST NOT)。
2. **承認タイムアウト**: ユーザーの応答待ちが 180秒 以上経過した場合は、処理を一時停止（保留）状態にして制御を返さなければならない (MUST)。

---

## 3. 動作安全性 (Safety Rules)
- **再帰・無限ループ防止**: 自律ループの最大イテレーション数を **5回** に制限し、それを超える自動実行は即座に停止して異常終了すること (MUST)。
- **エラー停止条件 (Stop Rule)**: サブプロセスやAPI呼び出しにおけるエラー、または書き込みの競合が連続して **5回以上** 発生した場合は、直ちにすべての自動処理を停止し、エラーログを要約してオペレータに報告すること (MUST)。
- **エコシステム整合性**: 実行は常に `uv` 仮想環境に準拠し、構造化ログは `structlog` を用いて標準エラー出力に出力すること (MUST)。
